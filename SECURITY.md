# Política de Segurança

## Versões Suportadas

Este projeto está em desenvolvimento como parte de um Trabalho de Conclusão de Curso (TCC).
Atualmente, apenas a versão mais recente disponível no repositório é considerada suportada para correções de segurança.

| Version         | Supported   |
| --------------  | ----------- |
| Mais recente    | ✅          |
| Versões antigas | ❌          |

---

## Reportar uma vulnerabilidade

Se você identificar uma vulnerabilidade de segurança neste projeto, por favor reporte de forma responsável.

Para relatar uma vulnerabilidade:

1. Descreva detalhadamente a vulnerabilidade encontrada.
2. Inclua passos para reproduzir o problema, se possível.
3. Informe o impacto potencial da vulnerabilidade.
4. Caso aplicável, sugira uma possível correção.

O reporte pode ser feito através de:

* **Issue privada no repositório**
* **Contato direto com o mantenedor do projeto**

Todas as vulnerabilidades reportadas serão analisadas e, quando confirmadas, corrigidas o mais rápido possível.

## Práticas de Segurança

Este projeto segue algumas práticas básicas de segurança:

* Validação de dados de entrada no backend
* Proteção de rotas autenticadas
* Uso de autenticação baseada em tokens
* Armazenamento seguro de credenciais (ex.: variáveis de ambiente)
* Dependências atualizadas para evitar vulnerabilidades conhecidas

---

## Isenção de responsabilidade

Este software foi desenvolvido para fins acadêmicos como parte de um Trabalho de Conclusão de Curso (TCC).
Apesar das boas práticas adotadas, ele pode não atender todos os requisitos de segurança necessários para uso em ambientes de produção.
